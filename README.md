# React App — Полнофункциональное приложение с маршрутизацией и API

## Описание

Учебное React-приложение, демонстрирующее:
- Маршрутизацию с React Router v6
- Работу с внешним API (JSONPlaceholder)
- Управление состоянием через Context API
- Добавление постов в избранное
- Адаптивный дизайн


## Структура проекта
- src/
- api/              # API слой
- components/       # Переиспользуемые компоненты
- context/          # Context API (глобальное состояние)
- pages/            # Страницы приложения
- App.jsx           # Корневой компонент
- main.jsx          # Точка входа
- index.css         # Глобальные стили

## Доработки
### Задание 0: Завершение функционала избранного

- **Страница избранного**:
  - Маршрут `/favorites` для отображения избранных товаров
  - Отображение списка с названием, описанием и метаинформацией
  - Кнопка «Удалить» для каждого товара в избранном
  - При удалении товар мгновенно исчезает из списка

- **Сохранение состояния**:
  - Избранное сохраняется в `localStorage`
  - Данные сохраняются между сеансами (работают после перезагрузки страницы)
  - Автоматическое обновление UI при добавлении/удалении товаров

### Задание 1: Авторизация пользователя
- **Форма входа**: страница `/login` с полями `email` и `пароль`
- **Валидация**:
  - Проверка формата email
  - Пароль: минимум 6 символов
  - Обработка ошибок и состояния загрузки
- **JWT-токены**:
  - Сохранение токена в `localStorage` при успешной авторизации
  - Сохранение данных пользователя для сессий
  - Автоматическая проверка токена при загрузке приложения
- **Защита маршрутов**:
  - Компонент `<ProtectedRoute>` проверяет наличие токена
  - Перенаправление на `/login` при отсутствии авторизации
  - Сохранение исходной страницы при перенаправлении на вход
- **Выход из системы**:
  - Кнопка «Выйти» в шапке приложения
  - Очистка токена и данных пользователя
  - Перенаправление на страницу входа

### Задание 2: Service Worker
- **Кеширование ресурсов**:
  - Файл `public/service-worker.js`
  - Стратегия `Cache First` для статики (HTML, CSS, JS)
  - Стратегия `Network First` для API-запросов
  - Кеширование основных ресурсов при установке
- **Офлайн-режим**:
  - Страница `public/offline.html` для офлайн-доступа
  - Сообщение «Вы в офлайне» при отсутствии соединения
  - Загрузка данных из кеша при недоступности сети
- **Обновление Service Worker**:
  - Событие `activate` очищает устаревшие версии кеша
  - Автоматическое обновление при изменении приложения
  - Корректная работа с несколькими вкладками

### Задание 3: Индикаторы статуса
- **Индикатор авторизации**:
  - Компонент `<UserMenu />` в шапке
  - Отображение имени пользователя после входа
  - Кнопки «Войти» (для неавторизованных) и «Выйти» (для авторизованных)
- **Индикатор сети**:
  - Компонент `<NetworkStatus />`
  - Красная плашка при отсутствии интернета
  - Сообщение: «Вы в офлайне. Некоторые функции могут быть недоступны»
  - Автоматическое исчезновение при восстановлении соединения
  - Отслеживание событий `online/offline`

## Дополнительные оптимизации

### Производительность
- `React.memo()` для компонента `Card`
- `useMemo()` для мемоизации контекста
- `useCallback()` для функций `toggleFavorite` и `isFavorite`
- `React.lazy()` + `Suspense` для ленивой загрузки страниц

### Доступность (Accessibility)
- ARIA-атрибуты для навигации (`role="navigation"`, `aria-label`)
- Семантические роли для меню (`role="menubar"`, `role="menuitem"`)
- Meta-теги для `viewport` и `description`
- Улучшенный контраст цветов
- Label для всех элементов форм

### Тестирование
- Настроен Jest + React Testing Library
- Тесты для основных компонентов
- Тесты навигации и авторизации
- Тесты работы с «избранным»

## Авторизация
```
1. Перейти на /login
2. Ввести email: test@example.com
3. Ввести пароль: 123456 (минимум 6 символов)
4. Нажать "Войти"
5. Перейти на /favorites (защищённый маршрут)
6. Нажать "Выйти" → перенаправление на /login
```

## Офлайн-режим
```
1. Открыть DevTools - Application → Service Workers
2. Проверить статус "activated and is running"
3. В Network выбрать "Offline"
4. Обновить страницу - загрузится offline.html
5. Включить сеть - приложение работает нормально
```
